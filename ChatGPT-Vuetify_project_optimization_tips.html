Your ProjectCard.vue is well-structured, but I see some areas for improvement regarding performance, accessibility, and responsiveness, especially with Vuetify components and asset loading. Here are my recommendations:

1. Optimize Image Loading for Netlify & Performance

✅ Use Lazy Loading

Your images are loaded immediately (:src="/medias/${project.title}/${image}"), which can slow down performance. Instead, use Vuetify's lazy-src and native loading="lazy":

<v-carousel-item v-for="(image, index) in project.images" :key="index"
    :src="`/medias/${project.title}/${image}`" 
    :lazy-src="`/medias/${project.title}/thumbnail_${image}`"
    loading="lazy" cover
    @click="openImage(`${project.title}/${image}`)">
</v-carousel-item>

✅ How it helps?

  • This ensures that only images that enter the viewport get loaded.
  • Use smaller thumbnails (thumbnail_${image}) as a placeholder before full-size images load.

✅ Serve Optimized Image Formats

Netlify serves assets efficiently, but using WebP instead of PNG/JPEG reduces file size.

  • Consider using Nuxt Image Module if moving to Nuxt later.
  • If using Vue only, preprocess images to WebP manually.

2. Accessibility (a11y) Enhancements

Vuetify follows Material Design but some elements need explicit accessibility improvements:

✅ Use alt Text for Images

<v-carousel-item v-for="(image, index) in project.images" :key="index"
    :src="`/medias/${project.title}/${image}`" 
    :alt="`${project.title} - image ${index + 1}`"
    cover>
</v-carousel-item>

✅ Ensure Buttons Are Focusable

  • Your v-btns have no aria-label or title, making them hard for screen readers:
<v-btn class="small-btn card-link-btn" rounded="0" variant="elevated" flat
    v-if="project.link" :href="project.link" target="_blank" 
    append-icon="mdi-open-in-new"
    aria-label="Visit live app"
    title="Visit live app">
    Visit live app
</v-btn>

3. Improve Responsiveness for Small Screens

You are using:

.v-card {
    aspect-ratio: 2/3;
    flex: 1 0 21%;
}

But on mobile screens, aspect-ratio will cause overflow issues. Instead:

@media (max-width: 600px) {
    .v-card {
        flex: 1 0 90%; /* Make cards take up more space on small screens */
        max-width: 90vw; 
    }
}

✅ Why?

  • Prevents cramped cards on mobile.
  • Ensures better UX for touch interactions.

4. Reduce Bundle Size & Load Time

Vuetify imports everything by default. Since you're using only specific components, tree-shake unused ones:

🚀 Replace:

import { defineComponent } from 'vue'

✅ With:

import { defineComponent } from 'vue'
import { VCard, VBadge, VChip, VTabs, VTab, VCarousel, VCarouselItem, VOverlay, VBtn } from 'vuetify/components'

Then, register only what’s needed:

export default defineComponent({
  components: { VCard, VBadge, VChip, VTabs, VTab, VCarousel, VCarouselItem, VOverlay, VBtn }
})

✅ Why?

  • Reduces bundle size.
  • Faster Netlify deploys.
  • Less unused JavaScript for mobile users.

5. Fix Badge Color Logic

Your getBadgeColor function has logic inconsistencies.

  • Current issue: If a project has no endDate, it's incorrectly marked sleeping after 180 days.
  • Fix: Change elapsed time condition:
getBadgeColor(project) {
    const today = new Date();
    const startDate = new Date(project.beginDate);
    const endDate = project.endDate ? new Date(project.endDate) : null;
  
    const elapsedDays = Math.floor((today - startDate) / (1000 * 60 * 60 * 24));

    if (elapsedDays > 180 && !endDate) return 'grey';  // Sleeping projects
    if (endDate && endDate < today) return 'rgba(3,180,255,1)'; // Ended projects
    if (startDate <= today) return 'rgba(0,200,0,1)'; // Active projects
}

✅ Why?

  • Prevents wrong classification of active projects as "sleeping".
  • More logical flow.
